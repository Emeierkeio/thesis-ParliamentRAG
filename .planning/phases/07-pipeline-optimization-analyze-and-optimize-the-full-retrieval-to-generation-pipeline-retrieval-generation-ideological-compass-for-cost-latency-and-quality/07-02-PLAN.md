Currently authority scoring (~line 157) runs THEN compass (~line 185) runs sequentially. Restructure so they run in parallel:

REPLACE the sequential authority block + compass block with:
```python
# Step 3+4: Authority scoring and Compass in PARALLEL
yield f"data: {json.dumps({'type': 'progress', 'step': 3, 'message': 'Calcolo authority scores e compass...'})}\n\n"

speaker_ids = list(set(e.speaker_id for e in evidence_list if e.speaker_id))

    None,
    lambda: services["authority"].compute_all_authority(speaker_ids, query_embedding),

compass_task = asyncio.get_running_loop().run_in_executor(
    None, services["ideology"].compute_2d_text_positions, evidence_dicts

```

Then keep the existing authority_scores extraction, expert computation, and compass data handling code AFTER the gather. The key change is that authority and compass now execute simultaneously instead of sequentially.

IMPORTANT: The expert computation (`compute_experts`) still happens AFTER authority completes (it depends on authority_scores), so keep it after the gather. The compass data formatting and SSE events also stay after the gather. Only the TWO run_in_executor calls move into the gather.

Preserve ALL existing SSE event emissions — do not remove any `yield` statements. Adjust the progress step numbering: steps 3 and 4 can be combined into a single progress message since they now run together.

3. **Client disconnect check**: The existing disconnect check between authority and compass can be removed since they now run together. Keep the disconnect check before generation (step 5).
