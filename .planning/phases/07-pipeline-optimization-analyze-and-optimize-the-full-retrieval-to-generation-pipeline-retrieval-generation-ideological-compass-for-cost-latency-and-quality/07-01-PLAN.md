**Part B -- benchmark_pipeline.py: Create benchmark harness**

Create `backend/scripts/benchmark_pipeline.py` as a standalone script that:

1. Loads `backend/evaluation_set.json` (7 topics)
2. For each topic, calls the full pipeline (retrieval + authority + generation) with the current model config
3. Captures per-topic metrics:
   - `latency_total_s`: wall-clock time for full pipeline
   - `latency_retrieval_s`: retrieval portion
   - `latency_generation_s`: generation portion
   - `cost_estimate_usd`: estimated from OpenAI response.usage (input_tokens * input_price + output_tokens * output_price)
   - `citation_count`: number of `[CIT:...]` or markdown link citations in output text
   - `parties_covered`: count of distinct parties with at least one citation
   - `section_completeness`: boolean -- all 10 parties have a section heading in output
4. Saves results to `backend/benchmark_results/{timestamp}_baseline.json`
5. Prints a summary table to stdout

The script must:
- Use `sys.path.insert(0, ...)` to add backend to path
- Import and instantiate services the same way the FastAPI app does (via `get_services()`)
- Use `asyncio.run()` for the async pipeline
- Accept `--output` CLI arg for output filename (default: auto-timestamped)
- Accept `--models` CLI arg to override model config (e.g., `--models analyst=gpt-4.1-mini,writer=gpt-4.1-mini,integrator=gpt-4.1-mini`)
- Price table embedded as dict: `{"gpt-4o": (2.50, 10.00), "gpt-4o-mini": (0.15, 0.60), "gpt-4.1-mini": (0.20, 0.80), "gpt-4.1-nano": (0.05, 0.20)}` (per million tokens)
- Create `backend/benchmark_results/` directory if not exists

Script structure:
