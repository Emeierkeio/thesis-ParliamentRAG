**Part A -- Create ner_channel.py**

Create `backend/app/services/retrieval/ner_channel.py` with a `NERChannel` class:

```python
"""NER entity retrieval channel.

Retrieves chunks that contain specific law references or person names
in their lawRefs/personRefs properties (populated by Phase 4 NER enrichment).
Only activated when query contains entity patterns (gated by engine).
"""
import logging
import time
from typing import List, Dict, Any, Optional

from ..neo4j_client import Neo4jClient

logger = logging.getLogger(__name__)


class NERChannel:
    """Retrieves chunks by NER entity match on lawRefs/personRefs."""

    def __init__(self, neo4j_client: Neo4jClient):
        self.client = neo4j_client

    def retrieve(
        self,
        entity_filter: Dict[str, list],
        top_k: int = 50,
        chambers: Optional[List[str]] = None,
    ) -> List[Dict[str, Any]]:
        """Retrieve chunks matching entity patterns.

        Args:
            entity_filter: {"laws": ["decreto 231", ...], "persons": ["Meloni", ...]}
            top_k: Max results
            chambers: Optional chamber filter

        Returns:
            List of result dicts matching sparse_channel format
        """
```

The Cypher query should:
1. Use `ANY(ref IN c.lawRefs WHERE ...)` for law matches -- use `toLower(ref) CONTAINS toLower($keyword)` for each keyword
2. Use `ANY(ref IN c.personRefs WHERE ...)` for person matches -- same toLower CONTAINS pattern
3. Combine law and person conditions with OR
4. Join to Speech, Speaker, ParliamentaryGroup, Session, Debate, Phase as in sparse_channel
5. Apply chamber filter if provided: `WHERE s.chamber IN $chambers` (same pattern as sparse_channel)
6. Limit to top_k results
7. Return result dicts in the SAME format as sparse_channel (evidence_id, chunk_text, speech_id, speaker_id, speaker_name, party, coalition, date, similarity, embedding, text, debate_title, session_number)
8. Set `similarity = 1.0` for entity matches (exact match -- RRF uses rank position, not raw score)
