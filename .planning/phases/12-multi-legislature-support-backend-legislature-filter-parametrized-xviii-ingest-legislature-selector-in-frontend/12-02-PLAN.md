<objective>
Add a `legislature` filter (int, default 19) to the timeline list endpoint so the /timeline page does not suddenly surface 2018-2022 sessions once XVIII data is ingested. Debate-detail and speaker-summary endpoints need NO filter — they are keyed by unique ids that already embed the legislature (`leg18_...` vs `leg19_...`).

Purpose: Keep the timeline scoped to the current legislature by default; expose the param so the frontend can switch later.
Output: `legislature` param on GET /api/timeline forwarded into get_sessions' Cypher WHERE, plus a Wave 0 source-inspection test.
</objective>
