## Spacing Scale

Declared values (must be multiples of 4). Source: Tailwind v4 defaults, confirmed by existing component usage in the project.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline element gaps |
| sm | 8px | Compact element spacing, stats row item gaps |
| md | 16px | Default card internal padding, search bar input padding |
| lg | 24px | Session card padding, section padding |
| xl | 32px | Gap between session cards |
| 2xl | 48px | Page-level top padding, major section breaks |
| 3xl | 64px | Page max-width gutter on wide viewports |

Exceptions:
- Touch targets on mobile: minimum 44px height/width for all interactive buttons (established project rule in `globals.css` `@media (pointer: coarse)`)
- Sticky header bar (search + chamber selector): 56px total height on mobile (16px padding top/bottom + 24px input height)
- "Back to top" button: fixed 40px x 40px, 16px from bottom-right viewport edge

**Accent (`--primary`) is reserved for:**
1. "Ask about this" button (primary variant)
2. Highlighted keyword match text background (`bg-primary/10 text-primary`)
3. Active/focused expand-collapse chevron
4. "Clear filters" button (outline variant with primary ring)
5. Speaker name link hover state underline

Accent is NOT applied to: card backgrounds, headers, most badges, or general text.

**Badge colors by semantic type:**
- Chamber badge (Camera/Senato): `variant="secondary"` — warm sand background, primary foreground
- Party badge: `variant="outline"` — border only, no fill
- Speaking role badge (Relatore, Presidente): `variant="secondary"` with `text-xs`
- Government member badge (shield icon): `variant="default"` — primary fill (Blu Istituzionale) with white text
- Parliamentary act tags: `variant="outline"` — border only

## Interaction Contracts

### Expand/Collapse — WAI-ARIA Disclosure Pattern

Source: locked in CONTEXT.md.

- `aria-expanded={isOpen}` on the trigger button
- `aria-controls={contentId}` on the trigger, matching `id` on content div
- `hidden={!isOpen}` on content (not CSS display:none — use `hidden` attribute for accessibility)
- `role="region"` on expanded content
- Keyboard: Enter and Space toggle; arrow keys navigate between sibling triggers
- ChevronDown icon rotates 180deg via `transition-transform duration-200` when `isOpen=true`
- Transition: `max-height` from 0 to auto via Radix `CollapsibleContent` (built-in animation)

### Search Bar — Debounced Auto-search

- 300ms debounce via `useRef<ReturnType<typeof setTimeout>>`
- No submit button — search fires automatically after debounce
- X clear button appears when input has value — clears input and resets results
- On search term change: reset `cursor=null`, `sessions=[]`, then fetch
- `aria-live="polite"` region announces result count after each search: "N sessions found" or "No sessions found"

### Date Range Presets

- Three `Button` components with `variant="outline"`, `size="sm"`: "Last week", "Last month", "Last 3 months"
- Clicking a preset sets both from/to date picker values and immediately triggers search
- Active preset highlighted: `variant="secondary"` (not outline) when active
- "Clear filters" appears only when a filter is active — `variant="ghost"`, resets all filters

### Infinite Scroll

- `IntersectionObserver` on a sentinel `div` at bottom of list, `threshold: 0.1`
- Shows `Loader2` spinner (16px, animate-spin) centered below last card while fetching
- `aria-live="polite"` announces "Loading more sessions" during fetch

### Speaker Summary — Lazy Load on Click

- Click SpeakerRow trigger → expand inline panel, fire `GET /api/timeline/speakers/{debateId}/{speakerId}`
- While loading: show "Loading summary..." text + `Loader2` inline spinner (12px)
- On success: replace with 2-3 sentence summary text (14px/400)
- On error: show "Summary unavailable" (12px, muted-foreground) — no retry button
- Already-loaded summaries are cached in component state; clicking again does not re-fetch

### "Ask about this" Button

- `variant="default"` (primary blue fill), `size="sm"`, icon: `ArrowRight` (14px)
- On click: navigate to `/` (chat page), pre-fill input with natural language question derived from debate metadata: "What positions were expressed in the debate on [debate.title] on [debate.date]?"
- Navigation uses Next.js `router.push('/?q=...')` with URL-encoded query param; chat page reads `?q` on mount

### Back to Top Button

- Fixed position: `bottom-4 right-4` (16px from viewport edges)
- Visible when scroll offset > 400px; hidden otherwise via `opacity-0 pointer-events-none` transition
- `Button` with `variant="secondary"`, `size="icon"`, `ArrowUp` icon (16px)
- `aria-label="Back to top"` — no visible text
- Click scrolls `window` to top with `behavior: 'smooth'`

## States & Skeleton Design

### Initial Loading (3-4 skeleton cards)

```
[Skeleton card — same height/layout as SessionCard]
  [Skeleton line — w-1/4 h-4 — date heading]
  [Skeleton line — w-full h-3 mt-2 — recap line 1]
  [Skeleton line — w-5/6 h-3 mt-1 — recap line 2]
  [Skeleton line — w-1/3 h-3 mt-3 — stats row]
```

Skeleton uses `bg-muted animate-pulse` (shadcn Skeleton component). Show 4 skeleton cards stacked with the same card border/padding as real cards.

### Empty State (no results)

Centered in the list area, vertically centered between sticky header and viewport bottom.

```
[Empty state — flex-col items-center gap-4 py-16]
  [CalendarDays icon — 40px, text-muted-foreground]
  [Heading — "No sessions found" — 16px/600]
  [Body — "Try adjusting your search or filters." — 14px/400, muted-foreground]
  [Button — "Clear filters" — variant="outline", size="sm"]
```

### Sessions Without Summaries

Inline within SessionCard where recap would appear:

```
[Italic text — "Summary not yet generated" — 12px, muted-foreground, italic]
```

No badge, no icon — just the muted italic note. Stats and debate titles still shown normally.

### API Error State

Shown in place of the timeline list. Appears only on hard fetch failures (network error or 5xx).

```
[Error state — flex-col items-center gap-4 py-16]
  [Alert component — variant="destructive"]
    [Alert title — "Unable to load sessions"]
    [Alert description — "An error occurred while loading the parliamentary timeline. Please refresh the page."]
```

No retry button — user refreshes. This matches the "minimal destructive actions" pattern in this phase.

### Debate Short (< 3 speeches — no AI summary)

Same DebateDetail layout but recap area shows:

```
[Text — "Not enough speeches to generate a summary." — 12px, muted-foreground, italic]
```

Phase list and speaker rows are still shown normally.

## Accessibility Contract

Source: locked in CONTEXT.md, supplemented by project-wide touch target rule (`globals.css`).

- WAI-ARIA Disclosure pattern on all expand/collapse triggers: `aria-expanded`, `aria-controls`, `role="region"`
- `aria-live="polite"` on: search result count announcement, loading states, empty state
- All icon-only buttons have `aria-label`: "Clear search", "Back to top", expand/collapse chevron uses parent button label
- Speaker name links: `aria-label="{speakerName} — view ranking"` to distinguish from bare text
- "Ask about this" button: descriptive enough — no additional `aria-label` required
- Keyboard navigation: Tab to session card trigger → Enter/Space toggles → Tab to first debate inside → Enter/Space to expand debate → Tab to "Ask about this" → Tab to first speaker → Enter/Space expands speaker summary
- Minimum touch target: 44px height/width on all interactive elements on pointer:coarse devices (enforced by `globals.css`)
- Color contrast: all text uses CSS variables from the institutional theme, validated to meet WCAG AA (primary `#1B3A5C` on white background exceeds 7:1)
