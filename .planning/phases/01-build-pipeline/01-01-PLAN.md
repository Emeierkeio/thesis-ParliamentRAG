Create `build/xml_parser.py` with class `StenograficoParser`:

1. **Constructor** — `__init__(self, config: BuildConfig | None = None)`. Store `self.ns = {'xhtml': 'http://www.w3.org/1999/xhtml'}`. NO Neo4j driver, NO `self.driver`. Accept optional BuildConfig for chunk params (default to BuildConfig()).

2. **Public API — `parse_xml_file(self, filepath: str) -> dict`:**
   - Parse XML with `xml.etree.ElementTree.parse(filepath)`
   - Extract session metadata (id, legislature, number, year, month, day, chamber, date as "YYYY-MM-DD" string)
   - Iterate `resoconto.findall('dibattito')` for debates, phases, speeches
   - For each debate: extract id (from `id` attribute), title (from `titoloDibattito` or first child text)
   - For each phase inside debate: extract title, assign `phaseType` via `classify_phase_type(title)`
   - For each speech (`testoXHTML` + `interventoVirtuale`): extract via `_parse_intervention()`
   - Call `merge_continuation_interventions()` on speeches per phase
   - **VOTE FIX**: Iterate `resoconto.findall('raccoltaVotazioni')` (NOT inside dibattito). For each `votazioni/votazione`, call `parse_vote()`. This fixes the zero-votes bug.
   - Call `_parse_act_references(root)` for debate-to-act links
   - Return `{"session": {...}, "debates": [...], "phases": [...], "speeches": [...], "votes": [...], "act_references": {...}}`

3. **`parse_vote(self, vot_elem, session_id: str, vot_index: int) -> dict`** — Port directly from `ingest_stenografici.py` `parse_votazione()` (lines 562-590). Return dict with keys: id, number, type, subject, present, voters, abstained, majority, inFavor, against, onMission, outcome.

4. **`_parse_act_references(self, root) -> dict[str, list[dict]]`** — New method. Navigate `root.find('metadati')`, then `metadati.findall('.//argomento')`. For each argomento, get `idDibattito` attribute. For each `atto` child, get `tipologiaAtto` and `codiceArgomento`. Return `{debate_original_id: [{"type": tipo, "code": codice}]}`.

5. **`_extract_speaking_role(self, testo_xhtml) -> str | None`** — New method. Find `nominativo` child, check if next sibling is `emphasis`. If so, strip and check against role prefixes: `('Ministro', 'Ministra', 'Sottosegretario', 'Sottosegretaria', 'Viceministro', 'Presidente del Consiglio', 'Relatore', 'Relatrice')`. Return role text or None.

6. **`_parse_intervention(self, elem, session_id, debate_id, phase_id) -> dict | None`** — Port from `parse_intervento()` (lines 476-560). Call `_extract_speaking_role()` and include `speakingRole` in returned dict. Call `preprocess_text()` (simplified version without alignment_map). Filter out president interventions and speeches shorter than `config.min_speech_length`.

7. **`merge_continuation_interventions(self, interventions: list[dict]) -> list[dict]`** — Port from lines 56-146. Merge consecutive speeches from same speaker when second starts with "...".

8. **`preprocess_text(self, raw_text: str) -> str`** — Simplified from `preprocess_text_with_alignment()`. Remove parenthetical content using `regex` library recursive pattern `\([^()]*(?:\([^()]*\))*[^()]*\)`. Remove speaker markers (PRESIDENTE, etc.). Return cleaned text. NO alignment_map return.

9. **`extract_text_from_element(self, element) -> str`** — Port from lines 203-226.

10. **Module-level function `classify_phase_type(title: str) -> str`** — implement the pattern mapping table from RESEARCH.md Pattern 7:
    ```python
    _PHASE_PATTERNS = [
        (re.compile(r'Dichiarazioni di voto', re.I), "vote_declaration"),
    ```
